# ResearchOS Technical Blueprint

ResearchOS is an autonomous AI research platform designed for multi-tenant scalability, real-time feedback, and high-quality insight synthesis.

## 🏗️ Architecture Overview

The application follows a modern full-stack architecture with a focus on real-time synchronization and agentic orchestration.

### 1. Frontend: Next.js 14 (App Router)
- **Framework**: Next.js with React 18.
- **Styling**: Tailwind CSS with a custom glassmorphic design system (Indigo/Cyan/Slate).
- **Authentication**: Clerk for secure, per-user tenant isolation.
- **State Management**: Real-time synchronization via Convex React hooks.

### 2. Backend: Convex
- **Real-time Database**: Provides reactive queries and mutations.
- **Serverless Functions**: Handles database logic and state transitions.
- **Multi-tenancy**: Strict isolation achieved via `tenantMembers` mapping and authentication helpers.

### 3. AI Orchestration: LangGraph & Groq
- **Engine**: LangGraph for stateful, multi-agent workflows.
- **LLM**: Groq (Llama-3-70b) for ultra-fast planning, reading, and analysis.
- **Agents**:
    - **Planner**: Generates dynamic research tasks from user objectives.
    - **Searcher**: Executes web navigation via Tavily API.
    - **Reader**: Summarizes search results into structured insights.
    - **Analyzer**: Synthesizes final reports with citations and trend detection.

## 🗄️ Database Schema

### Tenants & Users
- `users`: Core profile data (synced from Clerk).
- `tenants`: Workspace containers for projects and credits.
- `tenantMembers`: Links users to tenants with specific roles (Owner, Member).

### Research Data
- `projects`: Main containers for research missions (Status: idle, planning, researching, reading, analyzing, completed, failed).
- `tasks`: Individual agent tasks generated by the Planner.
- `reports`: Final markdown reports generated by the Analyzer.

### Analytics
- `usageLogs`: Chronological record of credit-consuming agent activities.

## 🛡️ Security Model
- **Token Verification**: Every request to Convex includes a Clerk `tokenIdentifier`.
- **Tenant Isolation**: Mutations and queries verify `tenantId` membership before processing data.
- **Environment Safety**: Secret keys (Groq, Tavily, Clerk) are stored in Convex/Vercel environments and nunca exposed to the client.

## 🧪 Scalability & Resilience
- **Timeouts**: Every agent step has a 60s timeout to prevent hanging jobs.
- **Error Recovery**: Search tasks fail gracefully, allowing the research to proceed with available data.
- **Optimistic UI**: Snappy project creation via Convex's optimistic update layer.
